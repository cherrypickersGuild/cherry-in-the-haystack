## Agent Types

### Reflex Agents
- 내부 추론 없이 입력을 직접 행동으로 매핑
- 빠르지만 단순하고 명확하게 정의된 작업에만 제한됨

### ReAct Agents
- 반복적인 추론과 행동을 루프로 교차 실행
- 추론 → 행동 → 관찰 → 추론 사이클

### Planner-Executor Agents
- 작업을 계획과 실행 단계로 분리
- 계획자가 계획을 수립하고 실행자가 작업을 수행

### Query-Decomposition Agents
- 복잡한 질문을 하위 질문으로 분해
- 각 하위 질문이 도구 호출을 트리거
- 결과를 최종 답변으로 종합

### Reflection Agents
- 과거 단계를 검토하여 실수를 식별하고 수정
- 반복적인 비판을 통한 자기 개선

### Deep Research Agents
- **기능**: 전문화된 데이터베이스와 학제 간 소스를 처리하는 고복잡성, 다단계 조사
- **한계**: 높은 컴퓨팅 비용, 지연 시간, 외부 데이터 소스의 취약성
- **사용 사례**: 깊이와 엄격함이 속도보다 중요한 장문의 전문가 수준 작업

## Tool Selection Strategies

### Standard Tool Selection
- 기본 모델이 제공된 도구 정의에서 선택
- 추가 인프라 없이 빠르고 직관적인 통합
- **도구 설명 모범 사례**:
  - 간결하고 설명적인 이름
  - 도구가 수행하는 작업에 대한 명확한 요약
  - 잘 정의된 입력 제약 조건

### Semantic Tool Selection
- 벡터 표현을 사용하여 도구를 인덱싱하고 검색
- 유사성 검색을 위해 도구 설명을 임베딩
- 대규모 도구 카탈로그에 유용

### Hierarchical Tool Selection
- 정확도 향상을 위해 도구를 그룹으로 구성
- 2단계: 그룹 선택 → 그룹 내 도구 선택
- 대규모 도구 세트에서 더 나은 확장성

### Tool Parametrization
- 모델 해석을 위한 매개변수 정의
- 특정 작업에 맞게 도구 동작 조정
- 타입 힌트와 설명이 모델을 안내

## Tool Topologies

### Single Tool Execution
- 하나의 도구를 선택하는 기본 패턴
- 단순 요청 → 도구 → 응답

### Parallel Tool Execution
- 여러 독립적인 작업을 동시에 실행
- 의존성 없는 작업의 지연 시간 감소

### Chains
- 의존적인 작업을 순차적으로 실행
- 한 단계의 출력이 다음 단계의 입력이 됨

### Graphs
- 복잡하고 비계층적인 흐름
- **조건부 엣지**: 런타임 결정에 따른 라우팅
- **통합 엣지**: 여러 브랜치 병합
- 추가 복잡성 오버헤드와 함께 최대의 유연성 제공

---

## Context Engineering
- 입력을 구조화되고 토큰 효율적인 컨텍스트 윈도우로 동적으로 조립
- **실천 방법**:
  - 관련성 우선순위 지정
  - 구조화된 포맷팅
  - 동적 조립
- 효과적인 에이전트 실행을 위해 메모리, 지식, 오케스트레이션을 연결

## Foundational Memory Approaches

### Context Window Management
- 토큰 제한이 있는 기본 모델 호출을 위한 작업 메모리
- 신중한 토큰 할당이 필요한 중요 리소스

### Rolling Context Window
- 윈도우가 가득 차면 가장 오래된 컨텍스트를 제거
- 메모리 관리를 위한 단순한 FIFO 접근 방식

### Traditional Full-Text Search
- 빠르고 정확한 과거 컨텍스트 검색을 위해 역인덱스 사용
- **BM25 스코어링**: 용어 빈도, 역문서 빈도, 길이 정규화로 구절 가중치 부여

## Semantic Memory

### Embeddings
- 사용 패턴을 기반으로 단어 의미를 포착하는 벡터 표현
- 의미적 유사성 검색의 기반

### Vector Stores
- 고차원 벡터 저장 및 검색에 최적화
- 임베딩된 콘텐츠 전체에 대한 빠른 유사성 검색

### RAG (Retrieval-Augmented Generation)
- **인덱싱**: 문서를 청크로 분할, 벡터 데이터베이스에 임베딩
- **검색**: 관련 정보를 위해 코퍼스 검색
- **생성**: 검색된 데이터와 모델 지식을 종합
- 정보에 입각한, 정확하고, 문맥적으로 관련성 있는 응답 가능

### Semantic Experience Memory
- 벡터 기반 상호작용 저장을 통해 세션 간 컨텍스트 유지
- 이전 상호작용에서 가장 일치하는 항목을 위해 컨텍스트 윈도우 예약

## GraphRAG

### Concept
- 향상된 검색을 위해 그래프 기반 데이터 구조 통합
- 흩어진 정보를 연결하는 데 있어 기본 RAG의 한계 해결

### Components
- **지식 그래프**: 엔티티와 관계를 그래프 형식으로 저장
- **검색 시스템**: 관련 하위 그래프 또는 클러스터를 위해 그래프 데이터베이스 쿼리
- **생성 모델**: 검색된 그래프 데이터를 일관된 응답으로 종합

### Building Knowledge Graphs
1. **데이터 수집**: 다양한 소스에서 정보 수집
2. **데이터 전처리**: 데이터 정제 및 표준화
3. **엔티티 인식**: 핵심 요소(사람, 장소, 조직)를 노드로 식별
4. **관계 추출**: 엔티티를 연결하는 술어를 엣지로 결정
5. **온톨로지 설계**: 카테고리와 관계를 스키마로 정의
6. **그래프 구축**: 온톨로지에 따라 노드와 엣지 생성
7. **통합 및 검증**: 엔티티 해결 및 테스트를 통해 정확성 보장
8. **유지관리 및 업데이트**: 지속적으로 새 데이터 추가 및 온톨로지 정제

### Semantic Triples
- 주어-술어-목적어 표현
- 지식 그래프 관계의 구성 요소

### Neo4j for Production
- 확장 가능한 배포를 위한 엔터프라이즈급 그래프 데이터베이스
- 그래프 작업을 위한 Cypher 쿼리 언어

### Knowledge Graph Benefits
- 멀티홉 순회
- 풍부한 관계 패턴 발견
- 다양한 애플리케이션에서 문맥적이고 정확한 응답

## Dynamic Knowledge Graphs

### Characteristics
- 실시간 정보로 지속적으로 업데이트
- 주기적 재훈련 없이 적응적 학습
- 현재적이고 관련성 있는 응답을 위한 실시간 데이터 통합

### Challenges
- **유지관리 복잡성**: 지속적인 데이터 유입으로 오류와 불일치 발생
- **리소스 집약성**: 업데이트, 검증, 유지관리에 상당한 컴퓨팅 필요
- **보안 및 프라이버시**: 사용자 데이터에 엄격한 규정 준수 필요
- **의존성 위험**: 과도한 의존은 인간의 감독 없는 의사결정으로 이어질 수 있음

### Mitigation Strategies
- 검증 메커니즘
- 확장 가능한 아키텍처
- 인간 감독 통합

## Long Context Windows
- Gemini와 GPT 같은 모델은 대규모 컨텍스트에 대해 검색 없는 생성 가능
- **트레이드오프**: 기존 검색 대비 더 높은 컴퓨팅 비용, 지연 시간, 정확성 보장

## Note-Taking Technique
- 모델이 답변 전에 입력 컨텍스트에 대한 메모를 작성하도록 프롬프트
- 사용 가능한 정보에 대한 명시적 추론 강제
- 구조화된 전처리를 통한 응답 품질 향상
