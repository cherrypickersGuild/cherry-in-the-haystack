## 1. 컴포넌트 목적

별도 페이지가 아닌 **플로팅 채팅 콘솔**. 카탈로그/대시보드 등 어떤 화면을 보든 화면 위에 항상 떠있고, 접고 펼 수 있다. 개발자 콘솔(DevTools)처럼 **항상 접근 가능한 에이전트 대화창**.

**쿼리 = 지식 구매**. 자유 대화가 아니라 **지식 구매 트랜잭션**:
1. 카탈로그에서 개념 확인 → 구매 버튼 클릭 → 콘솔에 자동으로 쿼리 전송
2. 또는 콘솔에서 직접 에이전트에게 지시
3. 구매(~20cr 일회성) 또는 팔로우(~5cr/업데이트) 선택 → 크레딧 결제
4. 구조화된 답변 수령 + 온체인 영수증 발급

PRD Journey 1 (심사위원):
> "화면에 Claude 에이전트가 Cherry KaaS에 질문한다 → 답변이 돌아온다 (출처별 에비던스, 품질 스코어) → 온체인 해시가 실시간으로 팝업 → Karma 티어 표시"

---

## 2. 사용 흐름 — 지식 구매 과정 시각화

이 페이지는 **사용자(Human) → 에이전트(AI) → Cherry KaaS(System)** 3자 간 지식 거래를 보여준다.

**진입 경로 2가지:**
- A. **카탈로그에서 진입**: 개념 모달의 구매/팔로우 버튼 클릭 → 콘솔 자동 팝업 + 해당 액션 실행
- B. **직접 진입**: Query 페이지에서 사용자가 에이전트에게 지시

```
사용자: "RAG를 구매해줘" (또는 카탈로그에서 구매/팔로우 클릭)
    ↓
에이전트: Cherry KaaS API 호출 (POST /v1/query, 20cr 결제)
    ↓
Cherry KaaS: 구조화된 답변 반환 + 온체인 영수증 기록
    ↓
에이전트: 사용자에게 구매한 지식 전달 + 프로비넌스 증명
```

화면에는 이 전체 과정이 **채팅 버블** 형태로 순차 표시된다.

### 4-2. 3자 대화 구조

메시지 타입 3종:

| 타입 | 발신자 | 정렬 | 시각적 특징 |
|------|--------|------|------------|
| **user** | 사용자 (Human) | 오른쪽 | 연회색 버블, "You" 라벨 |
| **agent** | AI 에이전트 | 왼쪽 | 보라색 아이콘, API 호출 메타 표시 |
| **cherry** | Cherry KaaS | 왼쪽 | cherry 좌측 보더, 구조화된 응답 |

### 4-3. 카탈로그 연동

카탈로그 모달에서 구매 버튼 클릭 시:
1. 플로팅 콘솔이 자동으로 펼쳐짐
2. 사용자 메시지가 자동 생성: "RAG를 구매해줘" 또는 "RAG를 팔로우해줘"
3. 에이전트가 바로 Cherry KaaS에 쿼리 실행
4. 카탈로그 화면은 그대로 유지 — 콘솔이 그 위에 오버레이

### 4-3. 사용자 지시 예시 (빈 상태에서 제안)

빈 화면 상태에서 예시 칩을 보여준다. **지식 구매 지시** 형태:

```
에이전트에게 지식 구매를 지시하세요:

[ "RAG 최신 evidence를 구매해서 학습해" ]
[ "네가 모르는 AI 개념이 뭔지 Cherry 카탈로그에서 확인하고 보고해" ]
[ "Chain-of-Thought 효과가 검증된 자료만 evidence로 구매해" ]
[ "임베딩 모델 비교 자료 summary로 먼저 받아봐" ]
[ "에이전트 아키텍처 패턴 중 production에서 쓰이는 것만 조사해" ]
[ "Fine-tuning vs RAG 어떤 게 나은지 evidence 기반으로 분석해" ]
```

핵심:
- **구매 행위가 드러나는 어투**: "구매해", "받아봐", "학습해"
- **액션이 드러나는 표현**: "구매해", "팔로우해"
- **카탈로그 기반**: Cherry에 있는 개념을 대상으로 지시
- 에이전트가 이 지시를 받아 Cherry KaaS API를 호출 + 크레딧 결제

### 4-4. 메시지 상세 — Agent 버블

에이전트가 Cherry KaaS를 호출하는 과정을 시각화:

```
┌─ Agent ──────────────────────────────┐
│ Cherry KaaS에 쿼리합니다...           │
│                                      │
│ POST /v1/query                       │  ← API 엔드포인트
│ {                                    │  ← 요청 body (접이식)
│   question: "...",                   │
│   action: "purchase",                │
│   budget: 20                         │
│ }                                    │
│                                      │
│ via MCP tool: query_knowledge        │  ← MCP 프로토콜 표시
└──────────────────────────────────────┘
```

- 상단: 에이전트의 행동 설명 (자연어)
- 중단: API 호출 상세 (코드 블록 스타일, 접이식 — 기본 접힘)
- 하단: 사용 프로토콜 (MCP tool 이름)

### 4-5. 메시지 상세 — Cherry 응답 버블

```
┌─ 🍒 Cherry KaaS ────────────────────┐
│                                      │
│ [Answer 텍스트]                      │
│                                      │
│ ▸ Evidence (3)          ← 접이식     │
│   ┌────────────────────────────┐     │
│   │ Chip Huyen — AI Engineering│     │
│   │ Retrieval-first mental...  │     │
│   │                            │     │
│   │ 💬 Hyejin Kim (Gold)      │     │  ← 큐레이터 코멘트 + Karma
│   │ "가장 접근성 좋은 입문 자료"│     │
│   └────────────────────────────┘     │
│   ┌────────────────────────────┐     │
│   │ ...                        │     │
│   └────────────────────────────┘     │
│                                      │
│ ★ 4.8 quality · 12 sources          │
│                                      │
│ ─── Provenance (1.5초 후 팝인) ───   │
│ ┌────────────────────────────────┐   │
│ │ 🔗 0xabc123...def456          │   │  ← 해시 링크 → Explorer
│ │ Status Network · Gasless       │   │
│ │ 20cr consumed (250 → 230)      │   │  ← 차감 before→after
│ └────────────────────────────────┘   │
└──────────────────────────────────────┘
```

- Answer: 구조화된 답변 텍스트
- Evidence: 접이식 (기본 접힘, 구매 시 전체 표시)
  - 각 evidence에 **큐레이터 이름 + Karma 티어 + 코멘트** 포함
- Quality + Sources: 한 줄 메타
- **Provenance 섹션**: 답변 표시 후 **1.5초 딜레이**로 나타남
  - "Recording on-chain..." 스피너 → 해시 + Explorer 링크 팝인
  - 크레딧 차감: `20cr consumed (250 → 230)` — before→after 명시
  - 이 영역에 배경색/보더로 **시각적 강조** (데모의 "와!" 순간)

### 4-6. 메시지 상세 — Agent 완료 버블

Cherry 응답 후 에이전트가 **사용자에게 결과를 전달**:

```
┌─ Agent ──────────────────────────────┐
│ Cherry KaaS에서 검증된 답변을         │
│ 받았습니다.                           │
│                                      │
│ 프로비넌스: 0xabc123...              │
│ 블록체인에 기록 완료.                 │
└──────────────────────────────────────┘
```

이렇게 3자 대화가 완성:
1. 사용자 → 에이전트 (지시)
2. 에이전트 → Cherry KaaS (API 호출)
3. Cherry KaaS → 에이전트 (구조화된 응답)
4. 에이전트 → 사용자 (결과 전달 + 프로비넌스 확인)

### 4-7. 프로비넌스 "와!" 순간 연출

답변이 나온 후 프로비넌스 영역의 타이밍:

```
0.0s — 사용자 메시지 표시
0.3s — Agent "Cherry KaaS에 쿼리합니다..." 표시
1.0s — Cherry 응답 (Answer + Evidence) 표시
1.0s — Provenance 영역: "⏳ Recording on-chain..." 스피너
2.5s — 스피너 → 해시 + Explorer 링크 팝인 (fade-in + scale 애니메이션)
3.0s — Agent 완료 메시지 표시
```

Provenance 팝인 시 시각 효과:
- cherry 색 보더 강조
- `animate-in zoom-in-95 fade-in` 으로 등장
- 해시 텍스트 monospace, cherry 색

### 4-8. 콘솔 헤더

```
🍒 Agent Console    ◆ Silver    250cr    [─] [X]
```

- Cherry 아이콘 + "Agent Console" 타이틀
- Karma 티어 뱃지 (Silver/Gold/Platinum)
- 크레딧 잔액 (쿼리마다 실시간 차감)
- [─] 최소화 (접힌 상태로) / [X] 닫기

### 4-9. 입력 바 (콘솔 하단)

```
┌──────────────────────────────┐ ┌──────┐ ┌─┐
│ 에이전트에게 지시하세요... │ │Purchase▾│ │▶│
└──────────────────────────────┘ └──────┘ └─┘
```

- 플레이스홀더: "에이전트에게 지시하세요..."
- 액션 드롭다운: Purchase 20cr / Follow 5cr per update
- Send 버튼 (▶)
- 크레딧 부족 시: 입력 비활성 + "Insufficient credits" 표시
- 카탈로그에서 진입: 개념명 + 액션(purchase/follow) 자동 설정 → 바로 실행

### 4-10. 목 데이터

카탈로그와 동일한 **9개 개념** 사용:
- RAG, Chain-of-Thought, Embeddings, Fine-tuning, Multi-Agent, Evaluation, Prompt Engineering, Agent Architectures, Semantic Search
- 각 개념별 키워드로 사용자 질문 매칭

사용자 입력을 에이전트가 **영어로 변환하여** Cherry API에 질문하는 것처럼 표시.
예: 사용자 "RAG 최신 정보 알아봐" → 에이전트 "Retrieval-Augmented Generation latest updates"

## 6. 후속 (API 연결 시)

- 목 search → `POST /api/v1/kaas/queries` fetch
- 목 provenance → 실제 chain-adapter 온체인 기록
- 에이전트 행동 → 실제 MCP tool 호출 결과
- Karma 티어 → 온체인 컨트랙트에서 조회
- 카탈로그 구매 버튼 → 콘솔에 이벤트 전달 (props callback 또는 context)
